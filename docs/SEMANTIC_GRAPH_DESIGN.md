# VeridianOS Phase 8 — Semantic Knowledge Graph Filesystem
## Design Specification

**Status:** Planned (after Phase 7 NES stabilization)
**Replaces:** Unix hierarchical file system (files + directories)
**Core Idea:** Store all data as *Objects* (entities) and *Relationships* (labeled directed edges)

---

## 1. Motivation

Traditional filesystems represent data as byte arrays located by hierarchical path strings:
```
/home/alice/projects/invoices/q4-2024-acme.pdf
```

This is an **accident of 1970s hardware** (disk sectors, directory inodes). It has no semantic
meaning. To find "all Q4 invoices for Acme," you must:
1. Know the exact directory path
2. Have an external database or grep-based search
3. Maintain naming conventions manually

**VeridianOS Semantic Graph** stores the same data as:
```
Object[uuid:invoice-q4-2024-acme] {
  type: Document,
  properties: { quarter: "Q4", year: 2024, amount: 15000 },
  blob: <PDF bytes as VMO>,
  edges: [
    (is_invoice_for, Object[uuid:company-acme]),
    (authored_by,    Object[uuid:person-alice]),
    (belongs_to,     Object[uuid:project-invoicing]),
  ]
}
```

Query: `find(type=Document, edge.is_invoice_for → company-acme, year=2024)`
→ Returns all matching invoices with zero path knowledge required.

## 2. Core Data Model

### 2.1 ObjectId
```rust
/// A globally unique, monotonically-increasing 64-bit identifier.
/// Generated by the kernel's atomic counter.
pub type ObjectId = u64;

pub const OBJECT_ID_NULL: ObjectId = 0;
```

### 2.2 ObjectType
```rust
#[repr(u8)]
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ObjectType {
    Blob        = 0,  // Raw bytes (replaces files)
    Document    = 1,  // Structured text/PDF/etc
    Image       = 2,  // Raster image data
    Code        = 3,  // Executable or source
    Config      = 4,  // Key-value configuration
    Contact     = 5,  // Person/org entity
    Project     = 6,  // Grouping concept
    Session     = 7,  // Login/auth session
    Agent       = 8,  // AI agent entity
    Custom(u8),       // Application-defined types
}
```

### 2.3 GraphNode (Kernel Object)
```rust
/// Kernel-managed node in the semantic graph.
/// Stored in the static GraphStore; lives until explicitly deleted.
#[repr(C)]
pub struct GraphNode {
    pub id:          ObjectId,       // Unique ID (never reused)
    pub object_type: ObjectType,     // What kind of entity this is
    pub vmo_handle:  u32,            // Handle to VMO containing blob data (0 = none)
    pub blob_size:   usize,          // Size of blob in bytes
    pub properties:  PropertyStore,  // Up to 8 key-value string pairs
    pub edges:       EdgeList,       // Up to 16 outgoing edges
    pub ref_count:   u32,            // Reference count (delete when 0)
    pub owner_pid:   u32,            // Creating process PID
}
```

### 2.4 Relationship (Edge)
```rust
/// A directed, labeled edge from one GraphNode to another.
#[repr(C)]
pub struct Edge {
    pub relationship: RelType,    // Label: what this edge means
    pub target:       ObjectId,   // The object this edge points to
}

#[repr(u16)]
pub enum RelType {
    Contains    = 0,   // A contains B
    IsPartOf    = 1,   // A is part of B
    CreatedBy   = 2,   // A was created by B (Person/Agent)
    IsVersionOf = 3,   // A is a version of B
    DependsOn   = 4,   // A depends on B (code/config)
    RelatedTo   = 5,   // Generic association
    IsInvoiceFor = 6,  // Document-specific
    BelongsTo   = 7,   // A belongs to project/group B
    Generates   = 8,   // Agent A generated B
    Custom(u16),       // Application-defined
}
```

### 2.5 Query Predicate
```rust
/// Simple predicate for graph queries.
/// Compose multiple predicates to filter the graph.
pub struct QueryPredicate {
    pub object_type: Option<ObjectType>,     // Filter by type
    pub property_key: Option<&'static str>,  // Filter by property key
    pub property_val: Option<&'static str>,  // Filter by property value
    pub edge_type: Option<RelType>,          // Filter by outgoing edge label
    pub edge_target: Option<ObjectId>,       // Filter: must have edge to this object
}
```

## 3. Syscall Interface

### sys_node_create (SYS_NODE_CREATE = 60)
```
Arguments:
  a0 = object_type (u8)
  a1 = blob_size (usize, 0 if no data)
  a2 = ptr to PropertiesInit struct (or 0)

Returns:
  a0 = new ObjectId (u64, low 64 bits)
       or negative error code

Side effects:
  - Allocates a GraphNode in the kernel's static GraphStore
  - If blob_size > 0, allocates a VMO (Virtual Memory Object) of that size
  - Returns a Handle to the node with full rights (READ|WRITE|LINK|DELETE)
```

### sys_edge_add (SYS_EDGE_ADD = 61)
```
Arguments:
  a0 = src_node_handle (u32)
  a1 = rel_type (u16)
  a2 = target_object_id (u64 low 32-bit portion, a3 = high 32-bit)

Returns:
  a0 = 0 on success, negative error code

Validation:
  - Caller must hold LINK right on src node
  - Target ObjectId must exist in GraphStore
  - Max 16 edges per node (returns -ENOMEM if exceeded)
```

### sys_node_write (SYS_NODE_WRITE = 62)
```
Arguments:
  a0 = node_handle
  a1 = src_ptr (user-space pointer to data)
  a2 = length
  a3 = offset into VMO

Returns:
  a0 = bytes written or negative error
```

### sys_graph_query (SYS_GRAPH_QUERY = 63)
```
Arguments:
  a0 = ptr to QueryPredicate struct (user-space, SUM must be set)
  a1 = ptr to output ObjectId array (user-space)
  a2 = max_results (usize)

Returns:
  a0 = number of matching ObjectIds written to output array
       or negative error code

Implementation:
  Kernel iterates all GraphNodes in static GraphStore,
  applies predicate filter, writes matching ObjectIds to user buffer.
  O(N) scan — Phase 9 will add hash indexes for O(1) type lookups.
```

## 4. Storage Backend

### Phase 8 (in-kernel static store)
```rust
const MAX_GRAPH_NODES: usize = 4096;

pub struct GraphStore {
    nodes: [Option<GraphNode>; MAX_GRAPH_NODES],
    next_id: AtomicU64,
    lock: spin::Mutex<()>,
}

static GRAPH_STORE: GraphStore = GraphStore::new();
```

### Phase 9+ (persistent on VirtIO block)
- Serialize GraphNodes to 512-byte sectors (fixed-size structs)
- Index: B-tree on ObjectId, hash table on ObjectType
- Write-ahead log (WAL) for crash safety
- Snapshots via copy-on-write VMO pages

## 5. Capability Model Integration

Each GraphNode is a kernel object:
```
Process A creates node → gets Handle{id=5, type=GraphNode, rights=RWL}
Process A sends handle to Process B via channel:
  → B gets Handle{id=7, type=GraphNode, rights=R} (read-only copy)
  → A retains Handle{id=5} with full rights
  → B can read blob but cannot add edges or delete
```

This is identical to how Zircon/Fuchsia handles VMO sharing, but for semantic graph nodes.

## 6. ASCII Diagram: Query Flow

```
User Process:                 Kernel:                    GraphStore:
                                                        [4096 GraphNode slots]
sys_graph_query(pred, buf)
    │                          │
    ├──────── ecall ──────────►│
    │                          │ trap_handler (scause=8)
    │                          │ syscall_handler(63, pred, buf, max)
    │                          │     │
    │                          │     ▼
    │                          │ sys_graph_query():
    │                          │   for node in GRAPH_STORE.nodes:
    │                          │     if node matches pred:
    │                          │       *buf++ = node.id
    │                          │       count++
    │                          │   return count
    │                          │     │
    │◄──── return count ───────┤◄────┘
    │                          │
    ▼
Read results from buf[]
(array of ObjectIds)
```

## 7. Implementation Plan

### Step 1: Types & Data Structures (kernel/src/semantic_graph/types.rs)
- [ ] ObjectId, ObjectType, RelType enums
- [ ] GraphNode, Edge, PropertyStore structs (all repr(C), no_std)
- [ ] QueryPredicate struct

### Step 2: Static Graph Store (kernel/src/semantic_graph/store.rs)
- [ ] `static GRAPH_STORE: Mutex<GraphStore>` with 4096 slots
- [ ] `alloc_node(type) -> ObjectId`
- [ ] `get_node(id) -> Option<&GraphNode>`
- [ ] `add_edge(src, rel, target) -> Result<(), Error>`
- [ ] `query(pred, out_buf) -> usize`

### Step 3: Syscall Integration (kernel/src/semantic_graph/syscall.rs)
- [ ] Register syscall IDs 60-63 in `syscall/numbers.rs`
- [ ] Implement all 4 syscalls with capability checks
- [ ] Wire into `syscall_handler` match arm

### Step 4: User-Space Library (user_programs/libsgraph/)
- [ ] C-compatible header (sgraph.h)
- [ ] Rust wrapper crate
- [ ] Test program: create nodes, add edges, query

### Step 5: Persistence (deferred to Phase 9)
- [ ] Serialize GraphStore to VirtIO block device
- [ ] Write-ahead log for crash safety
- [ ] B-tree index on ObjectId
